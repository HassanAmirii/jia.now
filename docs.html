Goal Tracker Documentation

A minimalist, developer-focused goal tracking system with AI-powered insights

Introduction

The Goal Tracker is a lightweight, single-page application designed for developers who want to track goals with precision and get AI-powered feedback. Unlike traditional habit trackers, this system focuses on:

  • Intentional goal setting (max 5 goals)
  • Nuanced progress tracking (5 states)
  • Pattern recognition through history
  • AI-powered honest feedback
  • Local-first data storage

Philosophy

Built by a developer for developers. The system embraces:

Less is more Deep work Patterns over streaks Radical honesty

The interface is intentionally minimal—no gamification, no streaks, no social features. Just you, your goals, and honest reflection.

Why Only 5 Goals?

Research in behavioral psychology suggests that humans have limited cognitive bandwidth for pursuing concurrent goals. The 5-goal limit is based on:

1. Cognitive Load Theory

Working memory can only hold 4-7 items simultaneously. Beyond 5 goals, tracking becomes overwhelming and counterproductive.

2. The Warren Buffett "25-5" Rule

List 25 goals, circle top 5—avoid the rest at all costs. Focus creates impact.

3. Developer Context

Engineering requires deep focus. Spreading attention across too many goals leads to context switching and shallow progress.

💡 The system enforces this limit not as a restriction, but as a forcing function for prioritization.

Progress Tracking States

State Value Description When to use
Locked In 100% Focused, intentional progress Deep work sessions, meaningful progress
Moving 75% Decent progress, not exceptional Solid day, but not breakthrough
Crawling 50% Minimal progress, but not zero Bare minimum done, distracted day
Struggled 25% Attempted but faced obstacles Blocked, frustrated, but tried
Skipped 0% Intentional break or miss Rest day, emergency, or reset

Why these specific states?

Traditional binary tracking (done/not done) misses nuance. A day where you're blocked but tried is different from a day you completely ignored a goal. These 5 states capture the spectrum of real human progress.

Check-in System

Check-ins happen daily and are stored with the format: goalId_YYYY-MM-DD

Calculation Logic

{ "Locked In": 100, "Moving": 75, "Crawling": 50, "Struggled": 25, "Skipped": 0 }

Progress Aggregation

Overall goal progress is calculated as the average of the last 7 check-ins (or all available if less than 7). This smooths out daily fluctuations and shows true momentum.

History & Pattern Recognition

The system maintains a history array for each goal, which enables:

  • Visual progress tracking over time
  • Pattern detection (e.g., always skipping on weekends)
  • AI context for more relevant feedback
  • Momentum calculations

History Structure

"history": { "goalId": ["Locked In", "Moving", "Skipped", "Crawling"] }

The array maintains chronological order (oldest to newest) and is capped at 30 entries to prevent bloat.

Complete Data Structure

{ "version": "2.0", "timestamp": "ISO-8601", "goals": [ { "id": "timestamp-based", "name": "Goal name", "timeframe": "Q1|Q2|Q3|Q4|Full Year", "importance": "high|medium|low", "why": "Motivation/reason", "progress": 0-100 } ], "checkins": { "goalId_YYYY-MM-DD": "state" }, "history": { "goalId": ["state1", "state2", ...] }, "reflections": { "q1": "Quarterly reflection", "q2": "", "q3": "" }, "chatMessages": [...], "userProfile": { "identity": "", "challenges": "", "support": "", "context": "", "lastUpdated": "" } }

Calculation Methods

Goal Progress

progress = sum(state_values) / count(checkins) * 100

Only includes check-ins from the last 7 days for current momentum.

Overall Progress

overall = average(all goal progresses)

Consistency Score

Calculated as: (Locked In count * 1.0 + Moving * 0.75 + Crawling * 0.5 + Struggled * 0.25) / total_days

Risk Assessment

AI analyzes patterns: if "Skipped" appears 3+ times in 7 days for high-priority goals, triggers warning.

  • Honest reviews - Brutally honest assessment of current progress
  • Risk identification - Spots patterns you might miss
  • Contextual advice - Based on your profile and history
  • Motivation - When you need it most

Prompt Engineering

The system constructs prompts using:

User context: {userProfile} Current goals: {goals} Recent history: {last7days} Chat history: {last5messages}

Local Storage

All data is stored locally in the browser's localStorage:

// Key: 'goalTrackerData' // No servers, no accounts, no tracking // Export/Import functionality localStorage.getItem('goalTrackerData') localStorage.setItem('goalTrackerData', data)

This ensures complete data ownership and privacy. Back up by exporting JSON.

API Reference

Core Functions

loadData() Loads from localStorage or initializes empty
saveData() Persists to localStorage
addGoal() Creates new goal (max 5)
updateCheckin() Records daily progress
calculateProgress() Updates goal percentages
updateHistory() Maintains history arrays

Extending the System

Want to add features? Here are common extension points:

1. Add New Progress States

// In updateCheckin() function const stateValues = { "Locked In": 100, "Moving": 75, "Crawling": 50, "Struggled": 25, "Skipped": 0 // Add new: "Crushing It": 110 };

2. Custom Reports

Add a reports section that generates weekly/monthly summaries using the history data.

3. Goal Categories

Extend goals with categories: "Learning", "Building", "Health", "Spiritual"
