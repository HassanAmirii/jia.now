JIA. Documentation

just f'in do it already — a no-fluff AI goal accountability manager

Introduction

JIA is a lightweight, single-page application built for people who are serious about their goals and want honest feedback — not dopamine hits from streaks or badges. It combines a structured daily check-in system with an AI advisor that actually knows your history.

  • Intentional goal setting (max 5 goals)
  • Nuanced progress tracking (5 states, not just done/not done)
  • Pattern recognition through history
  • AI-powered honest feedback via Groq
  • Local-first — your data stays in your browser

Philosophy

JIA is built on a few core beliefs:

Less is more Deep work over shallow hustle Patterns over streaks Radical honesty

The interface is intentionally minimal — no gamification, no streaks, no social features. Just you, your goals, and honest reflection. JIA will tell you when you're slipping, not congratulate you for opening the app.

Why Only 5 Goals?

The 5-goal limit isn't a bug — it's the whole point. Here's the reasoning:

1. Cognitive Load Theory

Working memory holds roughly 4–7 items at a time. Beyond 5 active goals, tracking becomes overwhelming and everything gets shallow attention.

2. The Warren Buffett "25-5" Rule

Write down 25 goals. Circle your top 5. Avoid everything else at all costs. The other 20 aren't "also-rans" — they're active distractions from what matters most.

3. Real Progress Requires Focus

Spreading effort across 10 goals means nothing gets done well. 5 forces you to prioritize, and prioritization is half the work.

💡 The limit is a forcing function. If you have a new goal and you're at 5, the question isn't "how do I add it" — it's "which current goal matters less?"

Progress States

State Value Description When to use
Locked In 100% Focused, intentional progress Deep work sessions, meaningful output
Moving 75% Decent progress, not exceptional Solid day, not a breakthrough
Crawling 50% Minimal effort, but not zero Bare minimum done, distracted day
Struggled 25% Attempted but hit obstacles Blocked, frustrated — but you tried
Skipped 0% Intentional break or miss Rest day, emergency, or reset

Why not just done/not done?

Binary tracking hides the truth. A day where you were blocked but still showed up is fundamentally different from a day you didn't even try. These 5 states let JIA (and you) see what's actually happening — and give the AI advisor real signal to work with.

Check-in System

Check-ins are logged daily per goal and stored with the key format: goalId_YYYY-MM-DD

State Values

{ "Locked In": 100, "Moving": 75, "Crawling": 50, "Struggled": 25, "Skipped": 0 }

Progress Aggregation

A goal's current progress is the average of its last 7 check-ins (or fewer if you haven't logged 7 days yet). This rolling window smooths out single bad days and shows true momentum rather than a lifetime average.

History & Pattern Recognition

Each goal maintains a chronological history array that powers the heatmap, trend chart, and AI context. It enables:

  • Visual effort heatmap across time
  • Daily effort trend chart (Chart.js)
  • AI pattern detection (e.g., consistently skipping on specific days)
  • Momentum calculations for the advisor

History Structure

"history": { "goalId": ["Locked In", "Moving", "Skipped", "Crawling", ...] }

The array is ordered oldest → newest and capped at 30 entries to keep localStorage lean. When the cap is hit, the oldest entry is dropped.

Complete Data Structure

Everything JIA knows about you lives in a single JSON object in localStorage under the key goalTrackerData:

{ "version": "2.0", "timestamp": "ISO-8601", "goals": [ { "id": "timestamp-based", "name": "Goal name", "timeframe": "Q1 | Q2 | Q3 | Q4 | Full Year | Monthly", "importance": "high | medium | low", "why": "Your motivation", "progress": 0–100 } ], "checkins": { "goalId_YYYY-MM-DD": "state" }, "history": { "goalId": ["state1", "state2", ...] }, "reflections": { "q1": "What made this week hard or easy?", "q2": "Which goal needs more attention?", "q3": "Any new ideas competing for focus?" }, "chatMessages": [...], "userProfile": { "identity": "", "challenges": "", "support": "", "context": "", "lastUpdated": "" } }

Calculation Methods

Goal Progress

Rolling average of the last 7 check-in values:

progress = sum(last_7_state_values) / 7

Focuses on recent momentum, not lifetime average.

Overall Progress

Simple average across all active goals:

overall = sum(all goal progresses) / goal_count

Consistency Score

Weighted effort score across all logged days:

consistency = ( (locked_in_count × 1.0) + (moving_count × 0.75) + (crawling_count × 0.5) + (struggled_count × 0.25) ) / total_days

Risk Detection

The AI advisor flags a goal as at-risk when Skipped appears 3 or more times in the last 7 days for a goal marked high importance. This surfaces in the Reflect tab feedback and in advisor responses.

AI Integration

JIA's AI advisor is powered by Groq running LLaMA 3.3 70B — accessed via a Netlify serverless function (netlify/functions/groq.js) that keeps your API key server-side and never exposes it to the browser.

What the AI can do

  • Honest reviews — brutally direct assessment of your current progress
  • Risk identification — spots patterns you might be rationalizing away
  • Contextual advice — tailored to your profile, goals, and history
  • Reflection feedback — analyses your weekly reflect answers and responds

Prompt Construction

Every AI request is built from live data, not static context:

System prompt includes: - User profile (identity, challenges, support style) - All current goals + importance + why User message includes: - Last 7 days of check-in history per goal - Last 5 chat messages (for continuity) - The user's actual question or reflection answers

🔒 Chat messages are routed through /.netlify/functions/groq. Your GROQ_API_KEY is an environment variable on Netlify — it is never bundled into the frontend code.

Local Storage

All goal data is stored locally in your browser. Nothing leaves your device except the messages you send to the AI advisor.

// Single key stores everything localStorage.getItem('goalTrackerData') localStorage.setItem('goalTrackerData', JSON.stringify(data)) // No accounts. No servers. No tracking.

Use Sync → Export to download a backup JSON file. You can re-import it on any device to restore your progress. Clearing browser data will wipe everything — export regularly.

API Reference

Core Functions

loadData() Loads state from localStorage, or initializes empty structure
saveData() Persists current state to localStorage
addGoal() Creates a new goal (enforces 5-goal max)
updateCheckin(goalId, state) Records today's progress state for a goal
calculateProgress(goalId) Returns rolling 7-day progress percentage
updateHistory(goalId, state) Appends to history array, trims at 30 entries

Extending JIA

1. Add New Progress States

In scripts/goals.js, extend the state values map:

const stateValues = { "Locked In": 100, "Moving": 75, "Crawling": 50, "Struggled": 25, "Skipped": 0 // e.g. "Crushing It": 120 };

2. Custom Reports

The history and checkins objects in localStorage have everything needed to generate weekly or monthly summaries. Hook into scripts/progress.js to add a new chart or export format.

3. Goal Categories

Extend the goal object with a category field: "Learning", "Building", "Health", etc. The data structure and sync logic will carry it through automatically.

Contributing

JIA is a personal project — fork it, adapt it, make it yours. The codebase prioritizes:

  • Readability over cleverness
  • Minimalism over feature bloat
  • Local-first over cloud dependency
  • Honesty over gamification

🔧 Vanilla JS, no frameworks, no build step. Clone the repo, run netlify dev, and you're live.
